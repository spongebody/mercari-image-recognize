# 模型测试页重构设计（Model Testing Page Redesign）

日期：2026-06-11
状态：已确认（经可视化方案对比后由用户逐项拍板）

## 背景与目标

现有「模型测试」页（`/evaluations`）由 5 个平级 tab 组成（发起 / 监控 / 校验 / 分析 / 多轮对比），把一条连续工作流切成孤岛：校验时看不到指标变化、导出入口与校验动线断裂、耗时数据已采集但从未汇总展示、「分析」tab 上线以来零使用。

重构目标：

1. 去碎片化——选中一个 run 后，状态、统计、校验、导出在同一页面连贯呈现；
2. 删除「分析」功能（前后端彻底删除），页面聚焦测试与校验；
3. 校验完成后顺手导出 CSV（格式不变），分析动作转线下；
4. 补齐统计面板（含平均耗时与阶段耗时）与强化多轮对比。

## 已确认的决策

| 决策点 | 结论 |
|---|---|
| 整体布局 | 单页测试台 + 新建测试抽屉；多轮对比为独立视图；不再有 tab |
| 统计面板 | 精简一行 5 卡；耗时卡展示 平均 / 分类阶段 / 商品数据阶段 |
| 多轮对比 | 结构化对比表（配置差异 / 质量 / 耗时三段），无图表依赖 |
| 导出 | 沿用现有 tab 分隔 CSV，入口移到校验区（及 run 头部） |
| 分析功能 | 前后端彻底删除（无历史数据损失：扫描确认 7 个历史 run 均无 analysis.md） |

## 1. 页面结构

页面只有两种主视图，由左侧栏切换：

```
┌──────────┬──────────────────────────────────────┐
│ 测试记录   │  视图一：测试台（默认，选中某个 run）       │
│  run 列表  │  视图二：多轮对比（点「⇄ 多轮对比」进入）   │
│ [＋新建测试]│                                      │
│ [⇄多轮对比]│                                      │
└──────────┴──────────────────────────────────────┘
```

- **左侧栏**：保留现有 run 列表（ID、状态、模型、复用配置按钮），底部固定「＋ 新建测试」主按钮和「⇄ 多轮对比」入口。
- **新建测试抽屉**：点「＋新建测试」从右侧滑出。表单字段与现状一致（数据文件、图片识别/分类/商品数据模型、推理程度、数量限制、语言），默认预填上一轮配置；run 列表里的「复用配置」改为打开抽屉并预填该 run 配置。提交后抽屉关闭、自动选中新 run 进入运行中状态。
- **「分析」删除范围**：前端 tab 与全部相关 UI；`PUT /api/v1/evaluations/{id}/analysis` 路由；`EvaluationRunStore.save_analysis`；详情接口响应中的 `analysis` 字段。历史 `analysis.md` 文件留在磁盘但系统不再感知。

## 2. 测试台（run 详情）的两种状态

同一页面按 run 状态分支渲染：

**运行中（pending / running）**：顶部 run 标题 + 状态 pill，主区域为大进度块（N/M、进度条、失败数、ETA），保持 2.5s 轮询；完成后自动切换到已完成状态，无需用户操作。

**已完成（completed / archived）**，纵向三段：

1. **头部**：run ID、状态 pill、模型配置一行（vision · category · productData · effort · 条数 · 语言）、操作按钮（复用配置 / 导出 CSV / 归档）。
2. **统计面板**（一行 5 卡，保存校验后实时刷新）：
   - 分类准确率（副行：复核后 X%）
   - 品牌准确率（副行：复核后 X%）
   - 平均耗时/条（副行：分类 Xs · 商品数据 Xs）
   - 待校验（副行：分类 N · 品牌 N）
   - 失败（副行：共 N 条）
3. **校验区**：过滤 chips（全部 / 分类错 / 品牌错 / 待校验，带计数）+ 批量操作 + 结果表格，占据剩余全部高度。

## 3. 校验交互

- **表格列收敛**：默认列 = 选择框、图、商品名、分类 原→AI、品牌 原→AI、分类判定、品牌判定、备注。AI分类Path、置信度、AI标题收进行内展开（点行内 ▸ 展开详情）。
- **键盘流**：↑↓ 移动焦点行、1/2/3 分类判定（正确/可接受/错误）、q/w/e 品牌判定保持不变；新增 Space 切换当前焦点行勾选（统一焦点与选择机制）；打标后自动跳到当前过滤器下的下一行。
- **Lightbox 内打标**：大图打开时 1/2/3、q/w/e 直接作用于当前行；←→ 翻图、Esc 关闭不变。
- **保存**：显式「保存校验」做成校验区顶部 sticky 条，显示「未保存 N 条修改」；切 run / 离开页面时若有未保存修改弹确认。
- **导出**：「导出 CSV」按钮位于校验区头部与 run 头部，调用现有 `/results.csv`，格式不变。

## 4. 多轮对比视图

- run 选择为胶囊点选，默认预选最近两轮已完成 run。
- 三段式对比表（行 = 指标，列 = run）：
  - **配置段**：图片识别/分类/商品数据模型、推理程度、数据量、语言；列间不同的单元格蓝色高亮。
  - **质量段**：分类/品牌准确率、复核后准确率、待校验数；最优值绿色高亮（准确率取最大，待校验取最小）。
  - **耗时段**：平均耗时/条、分类阶段、商品数据阶段；最小值绿色高亮。
- 沿用现有逐 run fetch + 会话内缓存逻辑（错误不缓存），纯 DOM 渲染。

## 5. 后端改动

1. **耗时汇总**：`app/evaluation/image_model_evaluation.py` 的 `_summary_bucket()` 新增 `avgTotalDurationS / avgCategoryDurationS / avgProductDataDurationS`（对每行已有的 `totalDurationS / categoryDurationS / productDataDurationS` 取平均，忽略缺失/非法值），随 summary 返回，单轮面板与对比表共用。
2. **删除分析**：移除 `runs.py` 的 `save_analysis`、`main.py` 的对应路由、详情响应的 `analysis` 字段。
3. 其余接口（创建、列表、详情、results、review、archive、results.csv）零改动。

## 6. 错误与边界

- 失败行 > 0 时统计面板失败卡变红，点击弹层显示 `errors.jsonl` 前 20 条摘要。
- 旧 run 缺耗时字段：耗时卡显示 `--`，对比表对应单元格留空，不报错、不参与最优值高亮。
- archived run：校验区只读（判定下拉禁用、批量按钮隐藏、无 sticky 保存条），导出仍可用。
- 多轮对比中数据集大小不同的 run 仍允许对比（配置段「数据量」行会以蓝色提示差异）。

## 8. 验证方式

- 后端：耗时汇总单元测试（含缺字段容错）；删除分析后全量跑现有测试，确认无残留引用。
- 前端：本地起服务用 Playwright 走完整动线（新建 → 运行 → 校验打标 → 保存 → 导出 → 对比），覆盖键盘快捷键、Lightbox 内打标、未保存确认、archived 只读。
